Universal access (Speed API from anywhere, no HTTPS reverse proxy)
====================================================================

Goal: the Capacitor app uses a user-entered base URL (e.g. http://203.0.113.10:3001).
The API must listen on a port reachable from the public Internet (or your chosen network).

Server (example: Ubuntu VPS / Droplet)
--------------------------------------
1. Clone this repo on the host.
2. Run:  sudo ./scripts/server-setup.sh --system && ./scripts/server-setup.sh --app
3. Copy scripts/speed-api.env.example to e.g. ~/.env.speed, set PORT=3001, export before start:
     export $(grep -v '^#' ~/.env.speed | xargs)
     export PATH="/path/to/repo/artifacts/api-server/.venv/bin:$PATH"
     cd /path/to/repo && pnpm --filter @workspace/api-server run start
   Or use ./scripts/server-setup.sh --print-systemd and systemd with EnvironmentFile=.

4. Firewall: allow the API port (e.g. sudo ufw allow 3001/tcp && sudo ufw enable).

5. In the app Settings, Server URL = http://YOUR_PUBLIC_IP:3001  (no trailing /api; the app adds /api).

iOS / Android apps
--------------------
- Info.plist includes NSAllowsArbitraryLoads so WKWebView can call http:// APIs you configure.
- AndroidManifest sets usesCleartextTraffic for the same reason.

Security
--------
HTTP is not encrypted on the wire. Anyone on the path can observe traffic. Prefer HTTPS when your
environment allows it; these settings exist so self-hosted HTTP endpoints work without nginx/Caddy.

App Store
---------
You may need to justify ATS / cleartext in review if you distribute publicly; self-hosted user URL
is a common pattern for developer / side-load builds.
